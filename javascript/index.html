Aprendiendo sobre los Objetos

Algo basado en grupo de informaciones o datos y sus efectos entre ellos mismos en vez de una serie de instrucciones. ~Cambridge dictionary.

Los objetos llevaran tus habilidades de programacion de Javascript a otro nivel. Estos son fundamentales para manejar codigos complejos, comprender el modelo-documento del navegador, organizar tus datos. Hasta son la forma fundamental en que estan empacadas muchas librerias de javascript.

Aqui esta el secreto: Los objetos de Javascript son solo colecciones de propiedades. Por ejemplo, digamos un carro. Un carro tiene propiedades:

  • marca: "chevy"
  • año: 1960
  • color: "rojo"
  • pasejeros: 4
  • convertible: false
  • mileage:20000
  • modelo: "Bel Air"

Piensa sobre un objeto como algo que almacena todos tus nombres y valores juntos, en otras palabras, engloba a las proiedades. Como se crea un objeto? Todo lo que hay que hacer es agregar el listado de propiedades y valores a una variable.

var chevy = {

marca: "Chevy",
model: "Bel Air",
year: 1960,
color: "red";
passengers: 2,
convertible: false;
mileage: 1021};

Ahora puedes agarrar el objeto y pasarlo alrededor, adquirir valor de el, cambiarlos, añadirle propiedades, o eliminarlo.

Hasta ahora hemos resuelto los problemas con una serie de variables declarativas, condicionales, for/while statements, y llamados a funciones. Esto es pensar de manera "procedimental"(me invente esa palabra..). Primero haz esto, luego aquello, y asi sucesivamente. Con la programacion orientada en los objetos pensamos en los problemas en base a los objetos. Los objetos tienen estados y comportamientos. Te permite liberar la mente y pensar en niveles superiores.

Como funcionan las propiedades

Ya tienes las propiedades del chevy empacadas en un objeto. Ahora que?
Puedes evaluar propiedades, eliminarlas, y en general, computar con ellas.

Como accesar una propiedad?

Para acceder a la propiedad de un objeto, empieza con su nombre
seguido de un punto, y luego el nombre de la propiedad.
Por ejemplo: fiat.mileage | chevy.color | civic13.model. Luego podemos usar las propiedades en cualquier expresion, ejemplo:

var miles = chevy.mileage;
if (miles < 2000) {
buyIt(); }

Como cambiar una propiedad?

Tu puedes cambiar el valor de una propiedad en cualquier momento. Todo lo que tienes que hacer es asignarle a la propiedad un nuevo valor. Por ejemplo: chevy.mileage = 10000;

Puedes extender los objetos en cualquier momento con nuevas propiedades. Para hacerlo solo necesitas especificar la nueva propiedad y darle un valor. Por ejemplo: chevy.needsWashing = true;

Como computar con propiedades

Computar con propiedades es muy simple: solo usa las propiedades como usarias cualquier variable(o cualquier valor). Por ejemplo:

if (chevy.year < 1980) {
classic = true;
}
for (var i = 0; i < chevy.passengers; i++) {
addPersonToCar();
}

Borra y Agrega

delete chevy.color | chevy.gpsIncluded = true;

Notas: Como una variable empaqueta un objeto

Sabemos que una variable es un contenedor y que empaqueta un valor.
Pero los numeros, las strings y los booleans son muy pequeños valores.
Que tal los objetos? Puede una variable empaquetar un objeto de cualquier
tamaño sin importar cuantas propiedades tenga?

  • Las variables no empaquetan objetos
  • En cambio ellas empaquetan una referencia de un objeto.
  • La referencia es como un indicador o puntero o una direccion hacia el objeto actual.
  • En otras palabras, una variable no aguanta un objeto per se, sino sostiene algo como punto o indicador.

Una variable primitiva representa el valor actual de una variable mientras que una variable objeto una manera de llegar hasta un objeto. Piensa sobre un objeto referente como otro valor de variable.

Una variable referente aguanta un valor que es un referente hacia los objetos.

Ejemplo de una variable primitiva:
var myCar = {...};
Un referente del carro va hacia la variable. El carro como objeto per se no va hacia dentro de la variable.

Comportamiento

Como darle vida a los objetos

Los objetos son activos. Los objetos pueden hacer cosas. Los perros no solo se sientan... ellos ladran, jugar a atrapar cosas. Un perro es un objeto tambien. Igual los carros: lo manejamos, lo parkeamos, lo ponemos en reversa, y los frenamos. Veamos como se ponen los objetos en accion:

var fiat = {
make = "Fiat",
model: "500",
year: 1957,
color: "Medium Blue",
passengers: 2,
convertible: flase,
mileage: 88000,
started: false,
this.start: function() {
started = true;
},
stop: function() {
this.started = false;
}, drive: function () {
if (this.started) { alert("Zoom zoom!);
} else {
alert("You need to start the engine first."); }
}
};

Hemos agregado una funcion directamente en el objeto. Todo lo que hay que hacer es agregarle una definicion-funcion a una propiedad. Asi mismo, las propiedades pueden tambien ser funciones. Notas que no le asignamos un nombre a la definicion de la funcion, solo usamos la keywork function seguida por el cuerpo. El nombre de la funcion es el nombre de la propiedad. A una funcion dentro de un objeto se le llama metodo.
